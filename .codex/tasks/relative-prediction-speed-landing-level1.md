## 3. 参考系预测轨迹设计

### 3.1 当前问题

`PhysicsEngine.predict_trajectory(probe, bodies, steps, dt)` 只返回探测器未来世界坐标。参考系下直接渲染这些世界坐标，会显示绝对轨迹，而不是相对参考天体的轨迹。

### 3.2 必须满足的计算方式

不能使用：

```text
reference_future = reference_current_pos + reference_current_vel * t
```

必须在预测时同步计算参考天体在未来每一步的真实位置。

推荐首版方案：

1. 在参考系下调用新的预测 helper，复制完整 `bodies`。
2. 对包含探测器和参考天体的完整系统逐步做真实物理更新。
3. 每步记录：
   - `probe_future_pos`
   - `reference_future_pos`
4. 渲染点转换：

```text
display_pos = current_reference_pos + (probe_future_pos - reference_future_pos)
```

这样预测轨迹在视觉上相对当前参考天体锚定，同时参考天体未来位置来自真实积分，而不是匀速估计。

### 3.3 验收

- 进入任意参考系后，预测轨迹不再显示绝对世界轨迹。
- 参考天体做明显曲线运动时，预测轨迹仍使用参考天体真实未来位置。
- 不修改真实 `bodies`。

## 4. 倍速控制设计

现有 `time_multiplier` 表示相对基础速度倍率。

新行为：

- `SLOW_HALF`：`time_multiplier = max(1/64, time_multiplier / 2)`
- `TIME_1X`：`time_multiplier = 1`
- `FAST_DOUBLE`：`time_multiplier = min(64, time_multiplier * 2)`

HUD：

```text
[<<] [1x] [>>]  4x
```

按钮含义：

- `<<`：放慢 2 倍
- `1x`：恢复原速
- `>>`：加速 2 倍
- 右侧显示当前具体倍率，支持 `1/64x`、`1/2x`、`1x`、`2x`、`64x`

键盘：

- 去掉旧 `5/6/7/8` 时间倍率快捷键。
- `Space` 暂停保留。
- 如未来需要新快捷键，另开任务。

## 5. 探测器降落设计

当前碰撞规则 `Probe vs any body -> probe_destroyed`。新规则：

- 探测器撞到非探测器天体时，不删除探测器。
- 探测器进入 `landed` 状态：
  - 位置固定在星体表面。
  - 速度随宿主天体速度，或简化为相对宿主静止。
  - 三角形尖端朝上。

首版数据建议：

- 不扩展 `BodyState`。
- 在 `main.py` 或未来 game 层维护 sidecar：

```python
landed_probes: dict[int, LandedProbeState]
```

字段：

- `host_body_id`
- `surface_normal`
- `offset_distance`

每帧物理更新后，对 landed probe 重新设置：

```text
probe_pos = host_pos + normal * (host_radius + probe_radius)
probe_vel = host_vel
```

碰撞处理需要避免物理层直接删除 probe。可选方案：

1. 在 `src/physics/collision.py` 改 Probe 碰撞事件为 `probe_landed`，并保持 probe active。
2. 或在 `main.py` 物理更新前后自定义检测探测器着陆，绕开删除逻辑。

推荐方案 1，因为规则属于碰撞规则，测试更清晰。
