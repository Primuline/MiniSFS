# MiniSFS Git 工作流规范

## 1. 分支与工作区

- Chief Agent 委派并行任务时，优先为每个任务创建独立 `git worktree`。
- 分支命名建议：
  - `feat/<name>`：新功能
  - `fix/<name>`：Bug 修复
  - `refactor/<name>`：重构
  - `test/<name>`：测试补充
  - `docs/<name>`：文档更新
- 不同子 agent 不应在同一个工作区并行修改。

## 2. 提交规范

提交信息格式：

```text
<type>: <简短描述>

<可选详细说明>
```

可用 type：

- `feat`：新功能
- `fix`：Bug 修复
- `refactor`：重构
- `test`：测试
- `docs`：文档
- `perf`：性能优化
- `chore`：杂项

## 3. 提交粒度

- 一次提交只做一件事。
- 不混合重构、修 Bug、新功能和格式化。
- 提交前运行与改动相关的测试。

## 4. 禁止行为

- 不直接向稳定分支推送，除非用户明确要求。
- 不提交调试打印、硬编码本机路径、敏感信息。
- 不使用破坏性 Git 操作清理用户改动。
