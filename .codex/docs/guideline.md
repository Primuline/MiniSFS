# MiniSFS Codex 通用开发守则

所有 Chief Agent 和子 agent 必须遵守本文件。

## 1. 沟通与汇报

- 每个子 agent 完成任务后向 Chief Agent 汇报：完成内容、关键决策、修改文件、测试结果、未完成项。
- 遇到接口签名、数据格式、模块归属不确定时，停止假设并向 Chief Agent 报告。
- 修改接口、目录结构、数据格式、用户操作流程后，同步更新相关 Markdown 文档。

## 2. 开发原则

- 小步改动：一次任务只解决一个清晰问题。
- 测试优先：核心物理、四叉树、碰撞、输入状态机等逻辑必须有测试覆盖。
- 使用标准库和 NumPy 已有能力，避免重复造轮子。
- 先保证正确、可读、可测试，再做性能优化。

## 3. 代码质量

- 可读性优先，变量名应表达意图。
- 复杂逻辑只在必要处添加解释“为什么”的注释。
- 面向接口编程，模块之间通过 `src/core/interfaces.py` 或稳定公开 API 协作。
- 对用户输入、文件加载、数值计算做边界检查。
- 物理计算必须防止除零、NaN、inf、无界增长等数值问题。

## 4. 项目约定

- 路径在文档中统一使用 `/`。
- 资源文件放在 `assets/` 下。
- 测试文件放在 `tests/` 下，命名为 `test_*.py`。
- 需求规格和验证清单放在 `docs/` 或 `.codex/tasks/` 下。
- Codex 工作规范放在 `.codex/docs/`，子 agent 角色说明放在 `.codex/agents/`。
